# Commit & Pull Request Guidelines

This folder is not currently initialized as a Git repository, so no local commit convention is available. Use concise, imperative commit messages if Git is added, such as `Add computer move tests` or `Refine input validation`.

Pull requests should include:

- A short summary of the change.
- Manual test notes, including the command run.
- Screenshots or terminal output only when user-facing behavior changes.
- Linked issues or context when applicable.
