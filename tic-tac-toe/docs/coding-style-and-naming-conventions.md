# Coding Style & Naming Conventions

Follow the existing simple Python style.

- Use 4-space indentation.
- Use `snake_case` for functions and variables.
- Use `UPPER_SNAKE_CASE` for constants such as `WINNING_LINES`.
- Keep functions focused on one job, for example input handling, winner detection, or move selection.
- Prefer clear type hints and short docstrings for public or reusable functions.

Avoid adding unnecessary frameworks or dependencies for core game behavior.
